Contributors

The following people have contributed ideas or code to QFTM. Where we have used published algorithms we have referenced one or more sources in the code itself and/or the documentation - these sources are not acknowledged in the list below.

  • Todd A. Ell has contributed ideas in various places including:
    1. code in subsref.m for structure indexing,
    2. ideas for faster convolution implemented in conv.m and conv2.m
  • The 'Mustard' convolution is based on work done with Hendrik de Bie and Todd A. Ell, and Todd has contributed to the wrap around convolution code in convw and convw2 which is used in the mustard function to provide compatibility with pointwise Fourier domain products.
  • Lukas Reichlin has contributed suggestions and code for Octave compatibility including:
    1. Changes to the char.m function to eliminate a nested function and to avoid display('text')
    2. Flat code for the mtimes.m function to overcome limitations on recursion depth.
